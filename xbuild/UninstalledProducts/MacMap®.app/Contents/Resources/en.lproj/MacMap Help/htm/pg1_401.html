XMap - Select

This function lets you edit a multicriteria request. The "Select" window is divided into three tabs:
Criteria
This first tab lets you specify the type of objects the request will affect, as well as the alphanumeric criteria. A search criterion is composed of three elements: a target calc or field, an operator and a value. Once the criterion is defined, clicking on the "+" button lets you add it to the list of criteria. The "-" button lets you delete a criterion. The "Open question" checkbox allows you to ask for a value every time the request is executed (useful for macros).
Relationships
These are the relationships (Geogs for geographic relationships) between objects. You can also select an object of a specific type contained in a list of area, selected or contrasted objects. The available geogs are:
Within A Distance Of
At A Distance Greater Than
Inside
Outside
Encompassing
Along
Starting Point
End Point
Nearest
Furthest
Starting From
Going To
Options
The options allow you to apply specific processing: replace the existing selection with the new one, add the new selection to the existing one, remove the selected objects from an existing selection, and include hidden objects.
It is also possible to contrast or hide the selected objects or the unselected objects.
Finally, you can also choose a macro that will automatically be run after the initial selection.
The "Save" button, visible when a call is made from the "Macros" box, allows you to store the request. It is then accessible in macro form.

Script Command 
<selector>
    <formula>
        <dbtype>Type name</dbtype>
        <dbselkind>Type of selection (1=Replace, 2=Add, 3=Subtract)</dbselkind>
        <dbusemasked>Include hidden objects (0 or 1)</dbusemasked>
        <dbaddtocontrast>Add selection to contrasts (0 or 1)</dbaddtocontrast>
        <dbaddtomasked>Hide Selection (0=Do not hide, 1=These ones, 2=The others)</dbaddtomasked>
        <dbsetnext>Optional, Macro identifier, in 4 characters.</dbsetnext>
        <!--Example of Geog (Optional)-->
        <lessthan>Less Than: object search distance</lessthan>
        <!--Example of formula-->
        <and>!--or: or--</and>
            <equal>!--or: higher, higherorequal, lower, lowerorequal, notequal, beginwith, endwith, contain--</equal>
                <dbfield>Field name</dbfield>
                <dbint>Value (dbfloat, dbchar, dbbool, dbdate, dbtime, dbquestion). Here you can pass a constant, for example fvarnow (now). For user constants, use fvaruser and pass the constant identifier as a parameter</dbint>
            </equal>
            <lowerorequal>
                <fcalcarea>!--Area. For user calcs, use fcalcuser and pass the calc identifier as a parameter--</fcalcarea>
                <dbfloat>Area value</dbfloat>
            </lowerorequal>
        </and>
    </formula>
</selector>
