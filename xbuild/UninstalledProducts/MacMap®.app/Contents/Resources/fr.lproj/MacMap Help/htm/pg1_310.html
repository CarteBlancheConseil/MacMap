XMap Importer des données

Permet d'importer des données attributaires externes dans une base MacMap®. Ces données peuvent êtres aux formats texte tabulé, csv, ou dbf. Après avoir spécifié le type dans lequel importer les données, on indique pour chaque champ du fichier l'action à appliquer:
- Ignorer pour ne pas prendre en compte le champ,
- Chercher pour définir une clef de recherche,
- Remplir pour transférer les données contenues dans ce champ,
- Créer puis remplir si un champ de même nom n'existe pas déjà dans la table cible.
Chaque champ du fichier externe doit donc être mis en relation avec un champ de la table cible.
L'option "Remplir un objet seulement" permet de détecter les cas ou une ligne du fichier concerne plusieurs objets.
L'option "Prendre en compte les objets masqués » permet d'inclure les objets masqués dans le traitement.
Les éventuelles erreurs d'import sont visibles dans le log Mac OS X (Application "Console"). Elles sont encadrées par les tags:
-- DATAJOIN REPORT BEGIN --
...
-- DATAJOIN REPORT END --
Elles sont également reportées dans un fichier texte d'extension".burp" situé au même niveau d'arborescence que le fichier importé.
Commande script 
<dataimport>
    <path>Chemin d'accès au fichier (POSIX)</path>
    <name>Nom du fichier (dbf, tabbed texte or csv)</name>
    <name>Nom du type</name>
    <bool>1 pour inclure les objets masquée, 0 sinon</bool>
    <bool>1 si la clef de recherche doit être unique, 0 sinon</bool>
    <!--Début de la description des opérations à effectuer sur les différentes colonnes, relativement à leur numéro-->
    <int>1(=Ignorer)|2(=Chercher)|3(=Remplir)|4(=Créer puis remplir)</int>
    <name>Nom du champ</name>
    <!--Cette séquence doit être répétée pour chacune des colonnes du fichier-->
</dataimport>
