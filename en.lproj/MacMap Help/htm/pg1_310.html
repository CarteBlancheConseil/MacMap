XMap - Import Data

This function lets you import external attribute data into a MacMap速 database. This data can be in tab delimited text, csv or dbf format. After selecting the type in which to import the data, you can specify the action to be applied for each field in the file:
- Ignore so as not to take the field into account,
- Search to define a search key,
- Fill to transfer data contained in this field,
- Create and then fill if a field with the same name does not already exist in the target table.
Each field in the external file must be linked to a field in the target table.
The "Fill only one object" option lets you detect the case where a line in the file involves several objects.
The "Include hidden objects" option lets you include hidden objects in the processing.
Potential import errors can be seen in the Mac OS X log ("Console" application). They are delimited by the following tags:
-- DATAJOIN REPORT BEGIN --
...
-- DATAJOIN REPORT END --
They are also reported in a text file with the ".burp" extension located at the same level as the imported file.
Script Command 
<dataimport>
    <path>File path (POSIX)</path>
    <name>File name (dbf, tabbed text or csv)</name>
    <name>Type name</name>
    <bool>1 to include hidden objects, otherwise 0</bool>
    <bool>1 if the search key must be unique, otherwise 0</bool>
    <!--Beginning of the description of operations to be carried out on the different columns, relating to their number-->
    <int>1(=Ignore)|2(=Search)|3(=Fill)|4(=Create then fill)</int>
    <name>Field name</name>
    <!--This sequence must be repeated for each column in the file-->
</dataimport>
