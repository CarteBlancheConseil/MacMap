XMap - Entry Constraints

This function lets you define entry constraints for a field. There can be three types of constraints:
- Natural: the type of constraint corresponds to the field type, for example you define integer values in an "Integer" type field.
- Class field: the constraints correspond to headings. In this case, the field value will correspond to the index of the constraint, as is the case for sub-type and color fields.
- Multiple values: only in "string" and "integer" fields. In the case of an integer, you can set up to 32 values and the field is coded bit by bit. In the case of a string, a series of 0s and 1s is obtained.
You cannot delete or modify a constraint for a table which contains objects.
