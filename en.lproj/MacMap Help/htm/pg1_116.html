XMap - Mif-Mid Import

Select and open one (or several) ".MIF" or ".TAB" file(s). The associated files are automatically opened. The fields contained in the Mif-Mid tables (or TAB) are automatically created if they do not exist in the corresponding type (see Import grid).
In order to remain geographically consistent, the imported Mif-Mid must be in the same projection as the database you are working with, or reprojected on import (see Import grid and Preferences).
The "Create type if necessary" option lets you integrate the imported data directly as MacMap速 tables, if the database does not contain a type with the same name. In this case, the import grid is not used, and the type created is automatically added in the current view.
Script Command 
<mitabimport>
    <path>Folder path (POSIX)</path>
    <name>File name</name>
    <bool>Create type if necessary</bool>
</mitabimport>
