XMap - Formula Editor

Lets you create or edit user-defined calcs.
The popup located at the top of the window lets you chose from among the elements forming a user-defined calc: Operators, Iterators, Functions, Constants, Fields, Calcs and Macros.
An additional popup is added when you select the "Fields" command: you can determine for which type you would like to obtain the list of fields.
The list located below these popups lets you display all of the elements available for a category.
The formulas are represented in tree form in the central drawing area. Double-click on this area to center the graph on the clicked point. Click on one of the elements in the formula to select it.
A selected element can be replaced by another element of the same category or erased using the "Erase" button.
Formulas are resolved from top to bottom, starting at the extremities of the tree.
Formulas are entered in Reverse Polish Notation, in other words placing first those elements which are resolved last.
To finish, you must specify the name and type of the returned value in the appropriate fields.
Operators:
Operators accept any two arguments. An operation between the two terms is carried out.
The currently available operators are:
and: logical and (boolean)
beginwith: string starting with another string (boolean)
contain: string containing another string (boolean)
div: division (floating)
endwith: string ending with another string (boolean)
equal: equality between 2 terms (boolean)
higher: greater than (boolean)
higherorequal: greater than or equal to (boolean)
lower: less than (boolean)
lowerorequal : less than or equal to (boolean)
minus: subtraction (floating)
modulo: modulo (integer)
mult: multiplication (floating)
notequal: different (boolean)
or: logical or (boolean)
plus: addition (floating)
pow: power (floating)
strcat: string concatenation (string)
xor: exclusive logical or (boolean)
Iterators:
Iterators accept two arguments; the first must be either a macro or left blank. In the first case, the iterator starts by running the macro and then performs an operation on the second term for the selection resulting from the macro. In the second case, the iterator uses the current selection.
The currently available iterators are:
average: average (floating)
first: take into account only the first object of the selection (the result depends on the second term)
last: take into account only the last object of the selection (the result depends on the second term)
max: maximum (the result depends on the second term)
median: median (floating)
min: minimum (the result depends on the second term)
product: product (floating)
single: return the NaN value if the selection includes more than one object (the result depends on the second term)
stddeviation: standard deviation (floating)
sum: sum (floating)
variance: variance (floating)
Functions:
Functions accept any argument.
The currently available functions are:
ceil: returns the smallest integer number which is not smaller than its argument (floating)
cos: returns the cosine of an angle in radians (floating)
exp: returns the exponential of its argument (floating)
fact: returns the factorial of its argument (floating)
floor: returns the largest integer number which is not greater than its argument (floating)
inv: returns 1/its argument (floating)
In: returns the Naperian logarithm of its argument (floating)
log: returns the base 10 logarithm of its argument (floating)
not: returns the opposite of its argument (boolean)
round: returns the round of its argument (floating)
sin: returns the sine of an angle in radians (floating)
sqr: returns the square of its argument (floating)
sqrt: returns the square root of its argument (floating)
strlen: returns the length of its argument as a string (floating)
strlwr: returns its argument as a lowercase string (string)
strsntfcupr: returns its argument in string format with the first letter in uppercase (string)
strupr: returns its argument as an uppercase string (string)
strwrdfcupr: returns its argument in string format with the first letter of each word in uppercase (string)
tan: returns the tangent of an angle in radians (floating)
Constants:
Fields do not accept arguments.
Standard constants currently available are shown in the Constant Box
Fields:
Fields do not accept arguments.
Calcs:
Calcs do not accept arguments.
Standard calcs currently available are shown in the Calc Box
Macros:
Macros do not accept arguments.

Examples
Population Density: To calculate the population density of an area type in a database in meters, you start by placing the "div" operator, then the "mult" operator on the lower branch of the former.
Next you place the "Population" field on the upper "div" branch, the "Area" calc on the upper "mult" branch and the "1000" constant on the lower "mult" branch to obtain a result in sq. km. Next you give this calc a name, and you specify the type for the return value (floating).
