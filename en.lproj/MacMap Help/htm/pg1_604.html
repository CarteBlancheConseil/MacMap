XMap - Presentation Box

The presentation box lets you view and edit document views. A view is composed of a series of styles which typically associate a type to a cartographic representation. A type can be represented as many times as necessary in the same view. It is also possible to define styles linked to external data rather than to a type, for example WMS style, which allows you to display rasters corresponding to the window display area through a request to a WMS server. These are called "virtual" styles. User-defined styles are created using editors called vDefs. The list of available vDefs is found at the end of this page.
Clicking on the style name displays a popup from which you can select another style.
Clicking on the left portion of a style lets you move it.
Double-clicking on the type name allows you to center the screen on the centroid of the type area.

The buttons found in the lower portion of the window let you do the following:
Move the selected style up/down.
Remove a style from the view.
Make the style selectable/unselectable. This option lets you prevent the tools from accessing certain styles and from displaying the selection for those styles.
Hide/show a style. This option lets you not show a style, without degrading the contents of the current view.
Style tracking. Turning this option automatically select the representation of the object that was clicked (regardless of the tool used). It is useful to quickly find the style that represents an object in complex views. Edit styles. The first section of the popup gives you access to the functions for managing styles. By choosing one of the other sections, you can add a style to the current view.
Edit views. The first section of the popup gives you access to the functions for managing views. By choosing one of the other topics, you can change the current view. The Set Reference Scale section lets you access the corresponding tool and set this parameter for each style of the current view.
