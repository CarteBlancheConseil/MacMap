MacMap Help

The Project File

The project file (".project") contains all of the references of a MacMapv3 database: It is the list of all of the "tables." This file is opened by MacMap.

It also contains all of the settings in common with the database:
- "Views": collections of "Styles" which define a cartographic representation of the database,
- The scales used in the database, the units of measure,
- "Macros": saved requests,
- All the global settings used by the different MacMap functions.
Type Files

These files (with a ".type" extension) are geographic object tables. Objects from the same table have the following in common:
- An attribute table structure by default: "Subtype", "Color", and "Direction" or "Flag" fields,
- An optional user attribute table structure: fields of type "String", "Integer", "Floating", "Boolean", "Date" or "Time"),
- A type of geometry (the "kind" of the type): "Point", "Line", "Area", "Text", or "Raster".

Each table can be used by different projects.

Each table contains settings used by the various MacMap functions: spreadsheet settings, vDef icons or raster images, ...
Default Fields

Name: A field of type string with a maximum of 50 characters.

Subtype: A field of type integer, with entry constraints. The default value is 1, with a heading at "Default Subtype". The user can add as many subtypes as necessary.

Color: A field of type integer, with entry constraints. The eight possible values are from 1 to 8 as follows: Black, White, Red, Green, Blue, Cyan, Magenta, Yellow.

Direction/Flag: This is a field of type integer, with constraints.
For area, line or raster objects, the Direction field corresponds to an orientation with the following values: -1 = orientation opposite from the direction of entry, 0 = no orientation, -1 = orientation in the direction of entry.
For text or point objects, the use of the flag field is not required. The possible values are 0 or 1.
Unit of Measure, Resolution and Projection

Unit: The unit in which distances will be expressed is specified when a new document is created. The unit can be meters, millimeters or kilometers.

Resolution: The resolution is also specified when a new document is created. It can be, for example, 0.5 meters.

Projection: Projection is implicit in MacMap, but it is still present in the interface. MacMap databases are necessarily formatted in a cartographic system and are, therefore, projected. It is possible to enter data outside of any repository, but it is better to wedge the data in a projection system (for example a UTM grid for your country). Some tools are available which let you change the projection of MacMap data.
Object States

Selected: The selection is a list of objects which can belong to different types. A selected object appears on screen surrounded by a rectangle with a thin, solid border in inverse video. A large number of MacMap functions operate on the current selection.

Contrasted: The contrasts are a list of objects which can belong to different types. A contrasted object appears on the screen surrounded by a rectangle with a thick raster border in inverse video. A number of MacMap functions operate on the current contrasts. The selection function, for example, uses contrasts to specify the objects which will serve as a reference in a search including distance criteria (at a distance greater than, within a distance of...).

Hidden: An object can be hidden. In this case, it no longer appears in the main window. Certain MacMap functions do not operate on hidden objects.
