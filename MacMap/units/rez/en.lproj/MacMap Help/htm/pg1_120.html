XMap - MacMap Text Import

Select and open one (or several) ".MMT" file(s). The fields contained in the file are automatically created if they do not exist in the corresponding type (see Import grid).
In order to remain geographically consistent, the imported file must be in the same projection as the database you are working with, or reprojected on import (see Import grid and Preferences).
The "Create type if necessary" option lets you integrate the imported data directly as MacMap tables, if the database does not contain a type with the same name. In this case, the import grid is not used, and the type created is automatically added in the current view.
Script Command 
<mmtimport>
    <path>Folder path (POSIX)</path>
    <name>File name</name>
    <bool>Create type if necessary</bool>
</mmtimport>
