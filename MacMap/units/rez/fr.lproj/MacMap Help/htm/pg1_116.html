XMap Import Mif-Mid

Sélectionnez un (ou plusieurs) fichier(s) ".MIF" ou ".TAB", et ouvrez. Les fichiers associés sont ouvert automatiquement. Les champs contenus dans les tables Mif-Mid (ou TAB) sont créés automatiquement s'ils n'existent pas dans le type correspondant (cf. Grille d'import).
Pour rester cohérent géographiquement, le Mif-Mid importé doit être dans la même projection que la base de donnée avec laquelle vous travaillez, ou re-projeté à l'import (cf. Grille d'import et Préférences).
L'option "Créer le type si nécessaire" permet d'intégrer les données importées directement sous forme de tables MacMap, si un type de même nom n'existe pas dans la base de données. Dans ce cas, la grille d'import n'est pas utilisée, et le type créé est automatiquement ajouté à la vue en cours.
Commande script 
<mitabimport>
    <path>Chemin d'accès au dossier (POSIX)</path>
    <name>Nom du fichier</name>
    <bool>Créer le type si nécessaire</bool>
</mitabimport>
