Aide MacMap

Le fichier projet

Le fichier projet (".project") contient toutes les références d'une base de données MacMapv3: c'est la liste de toutes les "tables". Ce fichier est ouvert par MacMap.

Il contient également tous les paramètres communs à la base de données:
- Les "Vues": collections de "Styles" qui définissent une représentation cartographique de la base de données,
- Les échelles utilisées dans la base, Les unités de mesures,
- Les "Macros": requêtes enregistrées,
- Tous les paramètres de portée générale utilisés par les différentes fonctions MacMap.
Les fichiers types

Ces fichiers (extension ".type") sont des tables d'objets géographiques. Les objets d'une même table ont en commun:
- Une structure de table attributaire par défaut: Champs "Sous-type","Couleur", et "Direction" ou "Drapeau",
- Une structure de table attributaire utilisateur optionnelle: champs de type "Chaîne de caractères","Entier","Flottant","Booléen","Date" ou "Heure"),
- Un type de géométrie (le "genre" du type): "Ponctuel", "Linéaire", "Surfacique", "Textuel", ou "Raster".

Chaque table peut être utilisé par différents projets.

Chaque table contient des paramètres utilisé par les différentes fonctions de MacMap: réglages du tableur, icônes ou trames des vDefs,...
Les champs par défaut

Nom : C'est un champ de type chaîne de 50 caractères.

Sous-type : C'est un champ de type entier, avec des contraintes de saisie. La valeur par défaut est 1, avec un intitulé à "Sous-type par défaut". L'utilisateur peut ajouter autant sous-types que nécessaire.

Couleur : C'est un champ de type entier, avec des contraintes de saisie. Les huit valeurs autorisées vont de 1 à 8 avec les intitulés suivant: Noir, Blanc, Rouge, Vert, Bleu, Cyan, Magenta, Jaune.

Direction/Drapeau : C'est un champ de type entier, avec contraintes.
Pour les objets surfaciques, linéaires, ou les rasters, le champ Direction correspond à une orientation avec les valeurs suivante: -1 = orientation inverse du sens de saisie, 0 = pas d'orientation, -1 = orientation dans le sens de saisie.
Pour les objets textuels ou ponctuels, l'utilisation du champ drapeau est libre. Les valeurs autorisées sont 0 ou 1.
Unité de mesure, résolution et projection

Unité : L'unité dans laquelle seront exprimées les distances est spécifiée à la création d'un nouveau document. L'unité peut être le mètre, le millimètre, ou encore le kilomètre.

Résolution : La résolution est également spécifiée à la création d'un nouveau document. Elle peut par exemple être de 0.5 mètres.

Projection : La projection est implicite dans MacMap, mais elle n'en reste pas moins présente. Les bases MacMapà sont nécéssairement en système cartographique, donc projetées. Il est possible de saisir des données en dehors de tout référentiel, mais il est préférable de caler ses données dans un système de projection (par exemple une grille lambert pour la France). Il existe déjà des utilitaires permettant de changer la projection de données MacMap.
Les états des objets

Sélectionné : La sélection est une liste d'objets pouvant appartenir à des types différents. Un objet sélectionné apparaît à l'écran entouré d'un rectangle en trait plein fin en inverse vidéo. Une grande partie des fonctions MacMap opère sur la sélection en cours.

Contrasté : Les contrastes sont une liste d'objets pouvant appartenir à des types différents. Un objet contrasté apparaît à l'écran entouré d'un rectangle en trait tramé épais en inverse vidéo. Une partie des fonctions MacMap opère sur les contrastes en cours. La fonction de sélection utilise par exemple les contrastes pour spécifier les objets qui serviront de référence dans une recherche incluant des critères de distance (A plus de, A moins de...).

Masqué : Un objet peut être masqué. Dans ce cas, il n'apparaît plus dans la fenêtre principale. Certaines fonctions MacMap n'opèrent pas sur les objets masqués.
